# 📄 ТЖ 3: Колдонуучу Баракчасы жана Төлөмдөр

Бул документ колдонуучулардын тажрыйбасын жана төлөм процессин аныктайт.

### 1. Тарифтер жана Төлөм Сурамы
- **Тарифти Тандоо:** Колдонуучу өзүнө ылайыктуу тарифти тандап, төлөмгө сурам жөнөтөт.
- **Төлөм Экрандары:**
    - Админдин картасынын **QR-коду** жана карта номери көрсөтүлөт (булар админ панелден башкарылат).
    - Төлөм бүткөндөн кийин колдонуучу тастыктоо үчүн **сүрөт (чек)** же **PDF** жүктөп бериши керек.
- **Маалымат Жөнөтүү:** Төлөм сурамы жөнөтүлгөндө, админге жана анын Телеграм ботуна маалымат барат.

### 2. Билдирүүлөр жана История (Notification History)
- **Башкы Бет (SMS/Alerts):** Колдонуучу киргенде, төлөмдүн абалы жөнүндө билдирүү ("Тастыкталды" же "Кабыл алынган жок") SMS формасында/индикатор катары көрсөтүлөт.
- **История Баракчасы:** Бардык билдирүүлөрдүн жана төлөмдөрдүн тарыхы тизмек катары сакталат. Бул жерден колдонуучу мурунку төлөмдөрүн жана алардын статустарын көрө алат.
- **Статус Башкаруу:** Төлөмдүн статустары: "Күтүлүүдө" (Pending), "Тастыкталды" (Confirmed), "Кабыл алынган жок" (Rejected).

### 3. Жеке Кабинет жана Дизайн
- **Ак Фон (Modern White):** Бардык баракчалар таза ак фондо, заманбап дизайн менен жасалат.
- **Адаптивдүүлүк:** Мобилдик телефондордо баардык функциялар (төлөм сурамы, билдирүүлөр) оңой колдонулушу керек (Responsive layout).
- **Слайддарды Башкаруу:** Колдонуучу өзүнүн мурунку презентацияларын көрүп, аларды кайра түзөтө алат.

### 4. Колдонуучунун Жолу (User Flow)
1. Колдонуучу тарифти тандайт.
2. Админдин QR-кодуна/картасына төлөйт.
3. Чегин жүктөө менен сурам жөнөтөт.
4. Админдин чечимин күтөт (статусу "Pending").
5. Админ тастыктаганда, колдонуучуга билдирүү келет жана премиум функцияларга доступ ачылат.
6. Бардык билдирүүлөр "История" бөлүмүндө сакталып калат.
