# 🤖 AI Презентация Түзүү Процессинин Документациясы

Бул документ AI менен презентация түзүү процессин камтыйт. Процесс кардардан маалымат алуудан баштап, аны Gemini AI моделине жөнөтүп, даяр слайддарды маалымат базасына сактаганга чейинки кадамдарды түшүндүрөт.

Процесс негизинен 3 этаптан турат:
1. **UI (Колдонуучу интерфейсинен маалымат алуу)**
2. **Server Action (Серверде иштетүү)**
3. **AI Генерациясы (Gemini API менен иштөө)**

---

## 1-кадам: Клиенттен Маалымат Алуу (UI)
**Файл:** `src/components/user/PresentationForm.tsx`

Бул кадамда колдонуучудан презентациянын негизги параметрлери алынат. 

Формада төмөнкүдөй маалыматтар чогултулат:
- **`prompt` (Тема же мазмун):** Колдонуучу презентация эмне жөнүндө болорун жазат (Мисалы: "Жасалма интеллекттин келечеги").
- **`slideCount` (Слайддардын саны):** 3төн 15ке чейинки санды тандоо мүмкүнчүлүгү. Дефолт боюнча: 5.
- **`tone` (Стиль/Тон):** Презентациянын стилин тандоо ("business" - Иштиктүү, "academic" - Илимий, "creative" - Чыгармачыл, "school" - Мектеп).

Колдонуучу "✨ AI менен түзүү" баскычын басканда, `handleSubmit` функциясы ишке кирет да, чогултулган маалыматтарды сервердик транзакцияга (`generateAndSavePresentation`) жөнөтөт. Ага чейин баскыч блоктолуп (disabled), загрузка (loading) состояниясы көрсөтүлөт.

## 2-кадам: Маалыматты Серверге Жөнөтүү (Server Action)
**Файл:** `src/lib/actions/gemini.ts`

Бул Next.js'тин `'use server'` функциясы. Бул файл клиенттен келген маалыматты алып, коопсуз түрдө AI моделине жана маалымат базасына өткөрүп берет.

```typescript
export async function generateAndSavePresentation(userId: string, prompt: string, slideCount: number = 5, tone: string = 'business') {
  // 1. AI функциясын чакыруу (Gemini)
  const presentationContent = await generateSlides(prompt, slideCount, tone)

  // 2. Даяр болгон структураны маалымат базасына (Supabase) сактоо
  const result = await createPresentation(
      userId,
      presentationContent.title || prompt,
      presentationContent.slides,
      'default'
    )
  
  // 3. Ийгиликтүү болсо, жаңы презентациянын ID'син кайтаруу
  return { success: true, id: successResult.data.id }
}
```

Бул жерде эгер генерация ийгиликтүү болсо, Next.js кэшти тазалайт (`revalidatePath`) жана кайра UI'га презентациянын ID'син кайтарат (ал боюнча колдонуучу `/editor/[id]` барагына багытталат).

## 3-кадам: AI (Gemini) менен Иштөө жана Промпт Түзүү
**Файл:** `src/lib/gemini.ts` (`generateSlides` функциясы)

Эң негизги логика ушул жерде жайгашкан. Маалыматтар AI'ге түздөн-түз текст катары эмес, абдан так структураланган системалык промпт (System Prompt) менен бирге жөнөтүлөт:

### А. Моделди жана API Ачкычты Даярдоо
Функция алгач базадан же чөйрөдөн (env) `GEMINI_API_KEY` алат жана `gemini-2.5-flash` моделин инициализациялайт.

### Б. Тонду (Стильди) Аныктоо
Колдонуучу тандаган `tone` параметри AI түшүнө турган инструкцияга айланат:
* **business:** "Профессионалдык, так жана бизнес стилинде..."
* **academic:** "Академиялык, илимий тилде..."
* **creative:** "Креативдүү, шыктандыруучу жана жандуу тилде..."
* **school:** "Жөнөкөй, түшүнүктүү жана кызыктуу тилде..."

### В. Системалык Промпт (System Prompt) Түзүү
AI моделге төмөнкүдөй катуу эрежелер (ДИЗАЙН ЭРЕЖЕЛЕРИ) берилет:
1. **Координаталар:** Слайддын өлчөмү `1920x1080` пиксел деп катуу белгиленип, элементтердин жайгашуусу (x, y, width кеминде 150px) абсолюттук мааниде берилиши талап кылынат. Промптто пайыз (0-100) колдонууга тыюу салынган.
2. **Дизайн:** Фондордун түрлөрү (gradient, solid color) берилип, текст менен фондун контрасттуу болушу эскертилет.
3. **Структура:** Берилген `slideCount` санына жараша: 1 Башкы слайд + N Негизги слайд + 1 Корутунду слайд болушу керектиги айтылат.
4. **Тил жана Формат:** Жооп **Кыргыз тилинде**, таза `JSON` форматында кайтарылышы катуу талап кылынат.

```json
// AI ушундай структурада жооп бериши керек (мисалы):
{
  "title": "Презентациянын аталышы",
  "slides": [
    {
      "title": "Слайд аталышы",
      "background": "linear-gradient(...)",
      "titleColor": "#ffffff",
      "elements": [
        { 
          "type": "text", 
          "content": "• Биринчи пункт", 
          "x": 150, 
          "y": 200, 
          "width": 1400,
          "fontSize": 36,
          "color": "#ffffff",
          "align": "left",
          "fontWeight": "normal"
        }
      ]
    }
  ]
}
```

### Г. Каталардын Алдын Алуу (Retry Logic)
Кээде AI API ашыкча жүктөлүп же жооп бербей калышы мүмкүн. Бул үчүн атайын `withRetry` функциясы колдонулат:
* Эгер `429 Too Many Requests` же `5xx` сервердик катасы келсе, система автоматтык түрдө бир аз күтүп (экспоненциалдык кечигүү менен), **3 жолуга чейин кайра аракет кылат**. Бул туруктуулукту (reliability) жогорулатат.

### Д. Жоопту Иштетүү
AI кайтарган тексттеги ````json ```` сыяктуу ашыкча белгилер тазаланып (`cleanJson`), `JSON.parse` аркылуу JavaScript объектисине айландырылат да, кайра Server Action'га (`generateAndSavePresentation`) кайтарылып берилет.

## Кыскача Архитектуралык Схема

1. **Колдонуучу (UI)** ➔ Тема, слайд саны, тон тандайт.
2. **Server Action** ➔ AI үчүн толук эрежелери бар `System Prompt` курат.
3. **Gemini 2.5 API** ➔ Текстти иштетип таза JSON структурасын берет (эгер ката чыкса, 3 жолу кайталайт).
4. **Server Action** ➔ JSON структураны алып `Supabase` базасына сактайт.
5. **Колдонуучу (UI)** ➔ Редакторго (`/editor/[id]`) багытталат, ал жактан слайддарды кол менен оңдой алат.
